蓝桥杯省赛学习第六周记录（常用的动态规划模型和背包）

发表于 2019-03-08 | 更新于 2019-03-21 | 次阅读

模型这个玩意儿真的美，啥题不会写上模板稍微改改都能过几个用例，不会也能妥妥的快速拿点分！

首先还是祭出动态规划的几条属性

  • 阶段：把求解的问题分为若干个阶段
  • 状态：每个阶段面临的各种客观条件
  • 决策 在每个阶段做出的不同操作
  • 状态转移方程 由i阶段的状态和决策决定i + 1阶段的状态
  • 策略 各个阶段决策的序列

下面的分析都会围绕着这几条属性展开

最大字段和

给出一组数字，要求求出其中子段和最大为多少

模板

分析

  • 首先输入如果全部为负数，则直接输出最大的（因为无论怎么加，和都只会更小）
  • 从第一个数字开始往后加，每加一步，将当前最大值记录下来
  • 一旦和小于0，则重新从0开始（一旦小于0，那我不如从下一个大于0的数字重新开始）
  • 只要不小于0都可以继续往后加（即使当前操作数字是个很小的负数），因为谁也说不准后面的数是什么，万一是个超级大的正数呢？因为已经记录了前面计算的最大值，所以不需要考虑，加就可以了）
  • 最后输出记录下的最大数字

我很笨，总觉得这个算法有点问题，后来研究了半天，发现确实是天衣无缝的🙃（所以您看我上面分析的那么啰嗦，最开始在这上面有点想不明白）

最大上升子序列

给出一段数， 问其中最长的上升子序列（不一定连续）长度是多少？
例如：2,1,5,3,6,4,6,3
他的最大上升子序列是：1,3,4,6 或 2,3,4,6

模板

分析

  • 状态：当前计算到的数字（以序列中的第几个数字为结尾）
  • 在每个状态下，我们遍历此状态前的每个状态所能取得的最大值和每个状态的对应位置的数字大小
  • 如果遍历到的状态数字小于当前状态，则当前状态的最大值，可能是遍历到的状态最大值 + 1（因为当前状态数字比遍历到的状态数字大，所以最长上升子序列长度 + 1）
  • 当把前面的状态遍历了一遍，自然可以的到最优的选择
  • 得到状态转移方程：dp[i] = max(dp[i], dp[j] + 1); i 代表当前状态， j代表遍历到的先前状态

可以看下面这种图理解，图中数字即上面给出的例子

这张图您需要动态的看，比如，dp[7] = 4 是怎么得到的呢？
遍历了1 - 6的dp值，发现状态dp 对应的数组值a[1] - a[6]都比a[7] 小。所以他们都可以作为决策，那么此时决策就是他们中间最大的dp值3
3 + 1 = 4;
结合上面模板，相信您能学会最大上升子序列的动态规划解法

最长公共子序列

给出两串数或者字符串，要求给出他们最长的公共子序列（不一定连续）

模板

模板给出的是字符串的，数字串用数组，同理。

分析

  • 状态：当前计算到的a串结尾位置和b串结尾位置分别是什么
  • 如果当前状态（a结尾字符与b结尾字符）相同，则可以将上一结尾字符相同状态的值 + 1（最长公共子系列长度 + 1）
  • 状态转移方程：dp[i][j] = dp[i - 1][j - 1] + 1;
  • 如果当前不相同，则当前状态下的最大值为 以第i - 1个字符结尾的a串 和 第j个字符结尾的b串 或 以第i个字符结尾的a串 和 第j - 1个字符结尾的b串 两个状态中的最大值
  • 状态转移方程：dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);

最小编辑长度

给出两个串，可以对第一个串中每个字符或数字增删改，问最少操作几次，两串可以相同？

模板

分析

  • 状态：当前计算到了a串的第几位，计算到了b串的第几位 对应二维dp[i][j]
  • 对当前状态，共有4中操作方式
  • 两个位置的字符相同，不需操作：该部分状态转移方程：dp[i][j] = dp[i - 1][j - 1]
  • 两个位置需要a改一个 ，该部分状态转移方程 dp[i][j] = dp[i - 1][j - 1] + 1，（上一步的最小操作数 + 1）
  • 两个位置差一位相同，需要a删一个 dp[i - 1][j] + 1;
  • 两个位置差一位相同，需要a加一个 dp[i][j - 1] + 1;
  • 对于边界：例如dp[0][j] 此时a串的状态是长度为0，所以此时的操作是j补，即为a串补上j个字符
  • 此时的状态转移方程：dp[0][j] = j; dp[i][0] = i;

01背包

给定一组物品,每种物品都有自己的重量和价值,现有一个背包,能承受的重量有限,在受限制的重量下,取若干物品,使得总价值最大
例如有5个物品，重量分别是[2,2,6,5,4] 价值分别是[6,3,5,4,6]

模板

分析

  • 状态：当前计算到第几个物品，此时背包容量剩余多少
  • 决策：是否将第i个物品放入背包，当前物品加入背包后容量没有超限则可以作为决策
  • 此时的状态转移方程为：dp[i][j] = max(dp[i - 1][j - c[i]] + w[i], dp[i - 1][j]);
  • 1.放入此物品，剩余体积减少[j - c[i]] 价值为放入上一个物品时的价值dp[i - 1] + 当前物品价值w[i]
  • 2.不放入此物品，剩余体积不减[j]不变 价值等于上个物品的价值dp[i - 1])
  • 当放入当前物品后容量超限:
  • 此时的状态转移方程为：dp[i - 1][j]
  • 即不放入此物品，剩余体积不减[j]不变 价值等于上个物品的价值dp[i - 1])

背包问题例题

最小背包-存钱罐

前面的例子都是求最大，最小怎么办？

分析

装入钱的总重量是存钱罐重量的前后差，每种钱只有一个，典型的01背包问题，只不过此题要求最小值，
这里就需要先给dp数组赋一个比较大的值，在使用memset的时候，理论上只能复制1或0,（或者其他任何高字节和低字节相同的数值）

0x3f3f3f3f是一个非常精妙，使用频率非常大的一个数字，（我在用文本编辑器时输入0x3后编辑器就自动联想到这个十六进制的数字）
为什么说这个数字精妙呢？首先他足够大，足以用作一个int内的无限大的数值来使用，同时，他还不是超级大，再加上另外一些比较大的数字时，不至于超出范围。同时，显而易见的，0x3f3f3f3f的任何高字节，都与低字节相同，可以用它结合memset()函数为数组赋值无限大

在为dp数组赋值无限大后还差一步，就是把第一次循环时（也就是分解到的对该问题的最小子问题的决策时）必定被选作最优选择的数值dp[0]赋0，这样只要是包含该决策的dp都会在该决策dp[0]的基础上增加数值，此时min函数就能起到作用，从无限大和已经做好决策正常数值的dp数值中选出正确的子问题决策.

认为超纲的题：平分娃娃与二进制枚举

分析

正常做这个题是不行的！它是一个典型的多重背包，但是用正常的手法有20%的数据超时了
解决的方法：需要用一个全新的便利方式：二进制枚举
这并不是纯粹的二进制枚举，只是差不多!
根本思想就是：我只往背包里放1，2，4，8…等2的移位运算后的到结果，
每次用移位运算而不是单纯的指针+1，目的就是快速找到当前状态下能放该物品的最大数量，因为中间的值很大程度上是不需要的，去遍历他们完全是浪费时间
当时这样难免会错过最后几个，比如当该萌值得娃娃数量为7，那么到4就退出了，5，6，7就还需要手动去遍历，所以在整个枚举后有多加了一次枚举补上剩下的几个数量的情况，具体看代码：
