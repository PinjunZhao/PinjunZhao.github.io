例题：浏览网页


分析：

首先是输入输出，VISIT和地址之间有空格，用scanf()不是不可以，麻烦，用cin简单，为了不超时
在main()函数开头加上std::ios::sync_with_stdio(false);
网页的BACK和FORWARD符合我们正常的浏览网页，用两个栈，一个存访问的地址，一个历史记录（回退之前的访问页面）
当BACK时，当前的访问从 访问地址栈 中弹出进入 历史记录栈，
当FORWARD时，历史记录栈 的栈顶元素出栈进入 访问地址栈
当VISIT时，清空历史记录栈，(因为VISIT后不允许再FORWARD)

著名的汉诺塔

汉诺塔（又称河内塔）问题是源于印度一个古老传说的益智玩具。
大梵天创造世界的时候做了三根金刚石柱子，在一根柱子上从下往上按照大小顺序摞着64片黄金圆盘。
大梵天命令婆罗门把圆盘从下面开始按大小顺序重新摆放在另一根柱子上(可以借助第三根柱子做缓冲)。
并且规定，在小圆盘上不能放大圆盘，在三根柱子之间一次只能移动一个圆盘。

当有三个圆盘时在柱子A上时：
先把最小的移动到柱子B上，再把中间的移动到C上 ，此时还需要再把小盘移动到C，把B柱子空出来
(简化为：借助B柱将 n - 1 个圆盘移动到C柱)

把大圆盘移动到 B

再借助A柱子把 n - 1 个圆盘移动到B柱（为了方便看，没有把最后一步画出来）

以上，无论有多少圆盘，都可分为三步
所以可以定义一个函数hanoi()

当这个n很大的时候，我们很难想出来他到底进行了多么复杂的运算，但当n为2的时候呢？
n = 2，只移动了一个圆盘，是不是就能看出一些道理？
而整个函数中不断的递归，就是将巨大的n转化为一个又一个的n = 2.
（将大步骤分解成小步骤，反正基本操作原理都一样）
